# Jevons' Paradox Applied to Software

> When the cost of making software drops, total software creation goes up dramatically.

## Content

Amjad Masad applies Jevons' Paradox — the observation that when the cost of a resource decreases, total consumption increases rather than decreases — to software development in the AI era.

**The mechanism:** When AI tools reduce the cost of building software from "hire a developer" to "describe what you want," people don't just build the same things cheaper — they build far more things they would never have built at all. The activation energy drops below the threshold that previously prevented creation.

**Evidence from Replit:**
- Real estate agents building custom back-office tools instead of buying ill-fitting SaaS
- CEOs building prototypes of next products instead of waiting for engineering bandwidth
- Marketing teams building competitive analysis applications
- Sales engineers spinning up customer-facing prototypes in minutes
- An 11-year-old building and deploying a complete app

**Implication for SaaS:** If anyone can generate custom software, the price you can charge for one-size-fits-all SaaS may collapse. The differentiator shifts from having built the software to the ability to rapidly iterate and generate ideas.

**Implication for product teams:** The bottleneck moves from production (building) to ideation (generating ideas worth building). Being generative — the ability to quickly generate new ideas — becomes the scarce skill.

## Guest Perspectives

### Amjad Masad ([[amjad-masad]])
Masad applies Jevons' Paradox to Replit's mission: as AI tools reduce the cost of building software to "describe what you want," total software creation explodes. Real estate agents build custom tools, CEOs build prototypes, 11-year-olds deploy full apps. The differentiator shifts from having built software to generating ideas worth building.

### Scott Wu ([[scott-wu]])
Wu's framing for Devin: software is "truly the shining example of Jevons' Paradox" — we have always managed as a society to find more and more things we want to build software for. His explicit application: as AI coding tools make programming 10x faster, the natural expectation would be 10x fewer engineers. But in practice, we'll build 10x+ more code, requiring even more engineers who can define architecture and specify exactly what they want. Wu cites the historical pattern: each abstraction layer (assembly → high-level languages → Python) expanded the total population of programmers rather than reducing it.
